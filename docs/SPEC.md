# システム要件定義書：ネット詐欺通報・検索プラットフォーム

**プロジェクト名**: Anti-Fraud Crowdsourcing Platform (仮称)
**バージョン**: 1.0
**作成日**: 2026年2月14日

## 1. プロジェクト概要

### 1.1. 背景と目的

近年増加するSNSやメッセージアプリを経由した詐欺被害を未然に防ぐため、一般市民からの情報を集約（クラウドソーシング）し、共有・検索できるプラットフォームを構築する。ユーザーは疑わしいURLやメッセージを検索・通報でき、運営側は集まったデータを分析して注意喚起を行う。

### 1.2. ターゲットユーザー

1. **一般ユーザー（閲覧者）**: 疑わしいURLを検索し、安全性を確認したい人。
2. **一般ユーザー（通報者）**: 詐欺情報を投稿し、社会に貢献したい人。
3. **システム管理者**: 通報内容を審査し、ステータスを更新・公開する運営担当者。

---

## 2. デザイン・UI/UX要件

### 2.1. テーマ設定

* **デフォルト**: **ライトモード**（白背景、黒文字）。信頼性と可読性を重視した公的機関のようなクリーンな配色。
* **テーマ切り替え**: ヘッダーまたは設定メニューに「ダークモード切替スイッチ」を配置。ユーザーのOS設定に追従、または手動での切り替えが可能。
* **アクセシビリティ**: Webアクセシビリティ規格（WCAG 2.1 / JIS X 8341-3）に準拠。コントラスト比の確保、スクリーンリーダー対応。

### 2.2. レスポンシブ対応

* PC、タブレット、スマートフォンの全デバイスで最適化されたレイアウトを提供する。

---

## 3. 機能要件：フロントエンド（一般ユーザー向け）

### 3.1. ホーム / 検索・一覧画面

* **検索機能**:
* 入力フォーム：URL、キーワード、メッセージ内容での検索。
* インクリメンタルサーチまたはオートコンプリート機能の検討。


* **一覧表示タブ**:
1. **注目（熱門）**: 閲覧数が多い順、または「いいね」が多い順。
2. **新着（近期）**: 最新の通報順。
3. **統計（統計）**: 統計ダッシュボードへのリンクまたは簡易表示。


* **フィルタリング**:
* 期間（今日、今週、今月）、カテゴリ（投資、フィッシング等）、ステータス。


* **案件カード表示**:
* サムネイル画像、タイトル、投稿からの経過時間。
* プラットフォームアイコン（LINE, FB等）。
* リスク判定バッジ（「詐欺確定」「高リスク」「調査中」）。



### 3.2. 案件詳細画面（Case Detail）

ユーザーが個別の通報内容を確認する画面。

* **ヘッダー**: タイトル、戻るボタン。
* **基本情報**:
* **証拠画像**: カルーセル形式または拡大可能な画像表示。
* **対象URL**: リンクとして表示（クリップボードコピー機能付き）。
* **内容詳細**: 通報時の説明テキスト。


* **ステータス・指標**:
* **発信元**: プラットフォーム名とアイコン（例: Facebook）。
* **現在の判定**: アイコン付きステータス表示（例: ⚠️ 高リスク）。
* **通報数**: この案件に対する重複通報の総数。


* **処理進捗タイムライン（Timeline）**:
* 垂直方向のタイムラインで以下を表示。
* `[日時]` `[アクション]`（例：ユーザー通報 → プラットフォームへ通知 → 詐欺認定）。


* **関連リンク**: 類似の詐欺案件へのリンク。

### 3.3. 統計ダッシュボード（Statistics）

収集データを視覚化する画面。

* **期間フィルタ**: 過去7日、30日、全期間などの切り替え。
* **キー数値（KPI）**: 指定期間内の総通報件数。
* **グラフ1：判定ステータス内訳（帯グラフ）**:
* 詐欺確定、高リスク、安全（非詐欺）、判断不能、処理待ち の比率。


* **グラフ2：詐欺手口の内訳（円グラフ）**:
* カテゴリ：製品/サービス、投資、ロマンス、求人、なりすまし、脅迫、寄付詐欺など。


* **グラフ3：プラットフォームの内訳（円グラフ）**:
* 媒体：Facebook, LINE, Instagram, Threads, Google, TikTok, Meta Audience Networkなど。



### 3.4. 通報（投稿）フォーム

* **入力項目**:
* 疑わしいURL（必須）。
* スクリーンショット画像（複数枚可）。
* 詳細説明（フリーテキスト）。


* **本人確認**:
* Cloudflare Turnstile による人間判定。
* honeypot + 送信速度制限 + IP単位レート制限によるスパム対策。


* **規約同意**: プライバシーポリシーへの同意チェック。

### 3.5. お知らせ・啓発（Announcements）

* **一覧**: 重要なお知らせ、最新の詐欺トレンド警告。
* **詳細**: リッチテキスト（画像、リンク含む）による記事表示。

---

## 4. 機能要件：バックエンド / 管理画面（管理者向け）

### 4.1. 通報管理

* **審査キュー**: 未処理の通報をリスト表示。
* **判定アクション**:
* ステータス変更（調査中 → 詐欺確定 / 安全 / 情報不足）。
* カテゴリ付与（投資詐欺、フィッシング等）。
* プラットフォーム特定。


* **タイムライン更新**: 「プラットフォーム事業者に削除依頼済み」などのログを追加。

### 4.2. コンテンツ管理 (CMS)

* **お知らせ管理**: 記事の作成、編集、削除、公開/非公開設定。
* **バナー管理**: トップページ等に表示する緊急告知バナーの管理。

### 4.3. マスタ管理

* **カテゴリ管理**: 詐欺の手口カテゴリの追加・修正。
* **プラットフォーム管理**: アイコンや名称の管理。

---

## 5. データ要件（データベース設計概要）

* **Users**: ユーザーID(UUID)、電話番号ハッシュ、作成日。
* **Reports**: 案件ID(UUID)、URL、タイトル、詳細、ステータスID、カテゴリID、プラットフォームID、閲覧数、リスクスコア。
* **Report_Images**: 画像URL、並び順。
* **Report_Timelines**: 案件ID、日時、アクション内容、担当者。
* **Stats_Daily**: 日付、カテゴリ別件数(JSON)、プラットフォーム別件数(JSON)、ステータス別件数(JSON) ※高速表示用。
* **Master_Data**: Categories, Platforms, Statuses.

---

## 6. 非機能要件

### 6.1. パフォーマンス

* **検索速度**: 数十万件のデータから1秒以内に検索結果を返すこと（インデックス最適化、必要に応じてElasticsearch等の導入）。
* **同時接続数**: スパイクアクセス（テレビ放送後など）に耐えられるオートスケーリング構成。

### 6.2. セキュリティ

* **個人情報保護**: 電話番号は不可逆なハッシュ化、または厳重に暗号化して保存。
* **データ通信**: 全通信のSSL/TLS化。
* **攻撃対策**: WAF（Web Application Firewall）の導入、DDoS対策、SQLインジェクション/XSS対策。

### 6.3. 運用・保守

* **バックアップ**: 日次バックアップおよびポイントインタイムリカバリ（PITR）の対応。
* **ログ監視**: エラーログ、アクセスログの保存と監視アラート。

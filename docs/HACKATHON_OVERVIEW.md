# ハッカソン企画概要書

## 1. 企画名

Anti-Fraud Extension Hackathon  
ネット詐欺通報・検索プラットフォーム拡張ハッカソン

## 2. 背景

本システムは、SNS・メッセージアプリ・不審なURLなどに関する詐欺情報を収集し、共有・検索できるプラットフォームです。  
今後の価値向上には、既存機能の改善に加え、通報体験、検索精度、分析、運営効率化、啓発機能などの観点から幅広い拡張アイデアを集め、素早く試作することが重要です。

## 3. 目的

- システムの新機能・改善案を幅広く発掘する
- 短期間で試作し、有望な案を今後の開発候補として具体化する
- エンジニア、企画、運営の混成で議論し、実運用に近い視点を取り込む

## 4. 開催概要

- 開催形式: 1日または2日間の集中型ハッカソン
- 対象者: エンジニア、デザイナー、PdM、運営担当、業務理解者
- チーム編成: 3〜5名程度の少人数チーム
- 成果物: 企画書、画面モック、プロトタイプ、簡易検証レポートのいずれか

## 5. テーマ例

- 通報しやすさを高めるUI/UX改善
- 詐欺判定や重複検知を支援する機能
- 検索、絞り込み、関連案件提示の高度化
- 管理画面の審査・オペレーション効率化
- 統計ダッシュボードや注意喚起機能の強化
- アクセシビリティ、多言語対応、モバイル最適化
- 外部連携やAI活用による新しい価値提案

## 6. 進め方

1. オープニングで課題背景、現行システム、評価観点を共有
2. 各チームで課題設定、アイデア発散、実装または試作
3. 最終発表でデモまたは企画提案を共有
4. 審査後、優秀案を今後の開発バックログ候補として整理

## 7. 評価観点

- 課題設定の妥当性
- ユーザー価値・社会的意義
- 実現可能性
- 新規性・発想の広さ
- デモや提案の分かりやすさ

## 8. 想定アウトプット

- 優先検討すべき機能案のリスト化
- 短期開発できる改善施策の抽出
- 中長期で差別化につながる新規構想の発見

## 9. 期待効果

本ハッカソンを通じて、現場課題に即した改善案と将来性のある拡張案を同時に収集し、本システムの価値向上と開発ロードマップの具体化につなげる。
